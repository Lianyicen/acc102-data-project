This project was designed to answer a simple but meaningful analytical question: which countries combine large market size and strong growth, and how can Python be used to communicate that clearly to a beginner user? My intended audience was international business students and beginner market analysts. I chose this audience because the assignment asked for a user-facing data product rather than a traditional essay. Instead of producing code only, I wanted to build a small project that could help a non-technical user compare countries in a simple and visually understandable way.

For the dataset, I used the built-in Gapminder dataset accessed through Python on 20 April 2026. I selected this dataset because it is structured, reliable enough for an educational task, and clearly related to economics and business discussion. It contains country-level indicators such as population, life expectancy, and GDP per capita across multiple years. I considered this dataset suitable because it allowed me to analyse both market size and development over time without needing a very advanced model. At the same time, it was manageable enough for me to understand every step of the workflow, which is important for this module.

In terms of Python methods, I used pandas for loading, cleaning, transforming, and summarising the data. I first checked the structure of the dataset and any missing values. Then I selected the variables most relevant to the project: country, continent, year, life expectancy, population, and GDP per capita. I dropped missing rows and standardised some basic fields. After that, I created a new variable, estimated total GDP, by multiplying population and GDP per capita. This was a simple transformation, but it made the analysis more useful because it gave me an approximate measure of market size. I then used groupby operations to compare continent-level patterns in 2007. Next, I merged 1997 and 2007 country data so I could calculate GDP per capita growth over a ten-year period. Finally, I used matplotlib to create charts and built a simple “market opportunity matrix” using size and growth thresholds.

The main outputs of the project were a structured notebook, several figures, and a GitHub-ready README. The continent comparison showed that Oceania had the highest average GDP per capita in the 2007 snapshot, while Europe also performed strongly among the larger multi-country regions. The market-size ranking showed that the United States was the largest market by estimated total GDP, followed by China, Japan, India, and Germany. When I looked at GDP per capita growth for countries with populations above 20 million, China and India were especially important because they combined strong growth with very large populations. In the final opportunity matrix, China, India, and Korea, Rep. appeared as the main priority markets because they met both the size and growth conditions. I think this result is useful because it goes beyond surface description and shows how two business-relevant dimensions can be combined into one simple screening framework.

There are also important limitations. First, the dataset ends in 2007, so the conclusions should not be used for real current investment decisions. Second, estimated total GDP is only a simplified calculation and does not replace official macroeconomic data. Third, this project does not consider exchange rates, inflation, regulation, sector-specific demand, or political risk, all of which matter in real market-entry decisions. If I improved this project in the future, I would replace the dataset with more recent World Bank or IMF data, add more variables, and possibly turn the notebook into a small interactive dashboard.

This task helped me understand how a data product should move from problem definition to user-oriented output. My own contribution included selecting the topic, deciding which variables mattered, choosing a clear user, and keeping the project simple enough to explain. I learned that a good project does not need a complicated model if the logic, workflow, and communication are strong. I also learned the importance of clear structure in both the notebook and the README so that another person can follow the project easily.
